Phase 1 — DATA / EVENTS

Two decisions live here: which MOMENTS the model learns on (the training regime), and which RAW MATERIAL feeds everything.

Training regime — which moments?
CHOSEN · era 1Strategy-fire eventslearn on entry-trigger moments: 151k fires, 3 baskets (MR/BO/HY), 5 yrs both instruments, honest costs; 5m/15m only (1m excluded — no edge over 5y, worst cost geometry)
NEXT ERA ⭐All-bar action corpusevery 5m bar gets counterfactual long/short/flat outcomes under frozen geometries — the model MINTS strategies instead of filtering them

How we build a corpus (the law of this phase) honest engine (reconciliation gate)→ real costs in→ temporal splits→ clean universe selection (no peeks)→ two corpora: execution vs contrast (≤20% weight)
Key finding: the corpus loses by cost geometry, not bad entries — gross ≈ break-even, costs ≈ the whole loss. New law: cost ≤ 10% of stop (ES stops ≥ 12.5pt, NQ ≥ 20pt) — designed in, not filtered after.

Phase 3 — LABEL

Same trades + different label = a different skill learned. Proven the project's biggest lever.

Tested — verdicts in
CHAMPIONWinner/loser tercilesonly label to pass fold-majority — but it's RELATIVE ranking: in 14/31 families "GOOD" = a smaller loss
OVERTURNEDFirst-touch (+1R before −1R)cleanly anchored it's barely learnable; early appeal was label contamination
KILLEDBinary win/lossunlearnable — coin flip
KILLEDR regressionlearnable, never profitable
KILLED as targetMFE/MAEMOST learnable, WORST money — selects volatility. Kept as forensic tool
Open menu — untested
NEW #1 ⭐Absolute action value Q(x, a)expected NET R per action {long, short, flat} — trade only when the lower-confidence bound > 0 after costs
CANDTriple-barrier + meta-labelingtextbook institutional method, never tried here
CANDRanking · distributional · multi-horizonorder the day's fires / full outcome range / per-horizon
CANDBenchmark-alignedlabel = contribution to P(funded ≤ 10d)
FOLLOW-UPEnsemble of labelsproven: different labels pick near-disjoint books
The label gauntlet pre-committed winner rule→ identical features/splits per arm→ judged on kept-book MONEY, never scores→ shuffle control + fold-majority
The MFE lesson (sticky): any label rewarding "how far price CAN run" secretly selects volatility — scores high, loses money. Learnability ≠ profitability.

Laws: beat the champion on identical data · complexity is the last lever · too-good = bug first (every spectacular PF in project history was a bug).

What execution does with a signal: veto or size, which timeframes, and the account policy. The phase the second-opinion review changed most.

Account-level (the benchmark layer)
PROMOTED ⭐Sizing / reset policy (Bellman)optimal risk-per-trade as % of remaining floor room under the $2k trail; resets valued as actions; now also risk-forecast-aware (vol-first sizing adopted from the research review)
CANDParallel accountsonly if genuinely independent — independence measured, never assumed
GUARDZero-edge controlevery policy must beat shuffled/no-edge paths under identical rules — a coin flip + bold resets "passes" 94% (lottery, not alpha)
The 5-day math (honest expectation table) $150 × 3/day: 5-day pass IMPOSSIBLE· E[profit] = target → P5 only ~55-62%· ~95% needs ≈6.5–7R/day (e.g. 70% WR · +2R · 6/day, net)
Law: "Policy can buy passage probability; it cannot certify alpha."

The gated sequence between "great backtest" and real money. Mostly linear — the options live upstream.

Law: forward performance is the only real judge — everything before gate 3 is rehearsal. After the vault read: no rescue tuning, ever.
